# earth_to_dxf

Google Earth画像や国土地理院画像を背景にして、簡易的に線をトレースしDXFへ出力するPythonツールです。

## 地理院タイル対応の設計メモ

- 地理院タイルは `z/x/y` 形式で取得します。標準地図と淡色地図はPNG、空中写真はJPGです。
- 現在の実装では、指定した緯度経度の中心タイルを基準に `3x3` タイルを取得し、1枚の背景画像へ結合します。
- タイル取得時はズームレベル `z` と緯度から `meters_per_pixel` を自動計算します。
- 地図種別は `std`（標準地図）、`pale`（淡色地図）、`seamlessphoto`（空中写真）を選べます。
- スクリーンショット画像の縮尺バーOCRは壊れやすいため、優先度は低めです。

## 住所検索

- 右側パネルの「住所検索」から住所入力欄を開けます。
- 入力した住所は Geolonia Community Geocoder 系の公開住所データで緯度経度へ変換します。
- 検索に成功すると、`gsi_settings.json` に `address` / `latitude` / `longitude` / `zoom` / `grid_size` / `tile_type` / `updated_at` を保存します。
- 住所検索後は、現在の `zoom` / `grid_size` / `tile_type` を使って国土地理院タイルを自動取得し、`config.json` と `scale.json` を更新します。
- 検索地点名は右側パネルの情報欄に表示されます。

## 地理院タイル使用時の注意

- 地理院タイル利用時は出典表示が必要です。
- 出典例: `出典：国土地理院ウェブサイト`
- 地理院のデータを加工してDXF化する場合は、国土地理院のデータをもとに加工した旨も記載します。
